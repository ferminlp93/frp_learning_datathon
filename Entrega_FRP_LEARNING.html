PATH_TO_IMAGES = '../Datos/Imagenes/'
image_paths = os.listdir(PATH_TO_IMAGES)
print(image_paths)

#Crear lista de imagenes
list_of_image_data = []


for im in image_paths:
	try:
		imag = PATH_TO_IMAGES + im
		image = Image.open(imag)
        #Convertir a monocroma
		image_grey = image.convert('1')
		hist_g = image_grey.histogram()
		width, height = image.size
		hist = image.histogram()
        #transformar la imagen a 50x50
		im = image.resize((50,50),Image.BICUBIC)
		arr = np.array(im)
		flat_arr = arr.ravel()
		im_g= image_grey.resize((50,50),Image.BICUBIC)
		arr_g = np.array(im_g)
		flat_arr_g = arr_g.ravel()
        #Extraer nombres
		aux=re.search(r"[^/]+$",imag)
		aux2=aux.group(0)
		aux3=re.search(r"[^_]*", aux2)
		ids=aux3.group(0)
		aux4=re.search(r"(?<=_)(.*)(?=_)", aux2)
		aux5=aux4.group(0)
		num=re.search(r"[0-9](.*)(?=_)", aux5)
		num2=num.group(0)
		final_name=ids+'_'+num2
        #Crear diccionario por imagen
		imdict = {
	    'name': final_name,
	    'height': height,
	    'width': width,
		'hist': hist,
		'hist_g': hist_g,
		'image3': flat_arr,
		'image1': flat_arr_g
	    }

		print(final_name,height,width)
        #aadir dictcionario de cada imagen a lista
		list_of_image_data.append(imdict)
	except Exception as e: print(e); continue

2.3 Natural Languaje Processing (NLP)¶

A continuacin, se tratarn las columnas con informacin en formato de texto abierto. Estas columnas de texto descriptivo pueden aportar gran cantidad de informacin por lo que se ha prestado especial atencin:

  1. HY_descripcion
  2. HY_distribucion

El anlisis de estas features se ha abordado basandose en la tncia de "bag of words", que consiste en anlizar las palabras existentes para convertirlas en features de forma que cada observacin tendr un valor que se corresponde con el nmero de veces que aparece esa palabra en el texto (descripcin o distribucin) de esa observacin. Haciendo uso de los mtodos de tf-idf o CountVectorizer se procede a la creacin de la "bag of words", aadiendo adems interacciones de palabras mediante los llamadas "n-grams" que agrupa palabras consecutivas posibilitando al modelo captar la informacin de estructuras de varias palabras que se suelen utilizar juntas.

Esta aproximacin resulta demasiado simplista y abusa de "fuerza bruta" resultando en modelos pesados con mucha informacin, aunque con una gran cantidad de ruido que impide la mejora del modelo. A consecuencia de esto se procede a emplear tcnicas de seleccin de features.

Se procede con la seleccin de palabras o combinacin de palabras (n-grams) que ofrezcan mayor correlacin con el TARGET. Para ello se utilizan tcnicas tipo SelectKBest o anlisis ms bsicos como la frecuencia total de aparicin de las palabras en el texto. Con esto, se ejecuta una primera aproximacin a la mejor solucin, pues ya se obtiene un grado de mejora sustancial en el modelo, sin embargo, no resulta suficientemente significativa.

Llegados a este punto, se opta por intentar ejecutar este mismo anlisis desde una perspectiva ms simple y manual. En conjunto con las palabras obtenidas con los mtodos anteriores, se opt por elaborar una lista de palabras clave que ms llaman la atencin de las personas que ven anuncios y trabajar con ellas. Para ello, se ha utilizado informacin obtenida de las propias pginas que ofertan inmuebles, donde se indican las palabras que habitualmente dan buenos resultados al incluirlas en la descripcin.
